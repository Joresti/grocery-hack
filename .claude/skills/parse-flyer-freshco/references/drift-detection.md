# Drift Detection — FreshCo

Run this diagnostic step **after switching to Grid View** but **before clicking "Load More"**. FreshCo's Tailwind-based class names are the most drift-prone of the three stores — Tailwind classes are generated by build tools and can change with any deployment.

## Structural Drift

### Selector hit rate

The skill depends on 8 selectors. Run this after the grid loads with initial products visible:

```js
(function() {
  var selectors = {
    gridContainer: '.grid.grid-cols-1',
    salePrice: '.text-red400.font-bold.text-body',
    regularPrice: '.line-through.opacity-50',
    saveBadge: '.bg-yellow150',
    sizeInfo: 'p.text-sm',
    productTitle: '.card-title span.text-grey900',
    sceneBadge: '#Scene_Card_exclusive_offers',
    gridToggle: 'button.icon-grid'
  };
  var report = {};
  var total = Object.keys(selectors).length;
  var hits = 0;
  for (var key in selectors) {
    var count = document.querySelectorAll(selectors[key]).length;
    report[key] = count;
    if (count > 0) hits++;
  }
  report._hitRate = hits + '/' + total;
  report._hitPct = Math.round((hits / total) * 100);
  return JSON.stringify(report);
})()
```

**Interpret:**
- **7-8/8 (88-100%)** — proceed normally. `sceneBadge` may be 0 if no Scene+ offers exist this week (that's content variation, not drift).
- **5-6/8 (63-75%)** — partial drift. Tailwind classes likely changed in a new build. Check which selectors broke. If `gridContainer` or `productTitle` are 0, extraction will fail completely. If `saveBadge` or `sceneBadge` are 0, only bonus features are affected.
- **0-4/8 (<50%)** — hard drift. FreshCo (Sobeys/Empire) likely redesigned or rebuilt the site. Stop and investigate.

**FreshCo-specific risk:** Tailwind classes like `.text-red400` and `.bg-yellow150` are tied to a design token config. If Sobeys renames their color palette (e.g., `red400` → `red-500` or `danger`), multiple selectors break simultaneously. If you see 3+ color-based selectors fail at once, check whether the color naming scheme changed rather than investigating each selector individually.

### DOM path fingerprint

Expected structure from grid to card:

```
.grid.grid-cols-1 > div (wrapper) > div.rounded-xl (card) > div.p-4 (info section)
```

```js
(function() {
  var grid = document.querySelector('.grid.grid-cols-1');
  if (!grid) return JSON.stringify({drift: true, reason: 'No grid container found'});

  var firstChild = grid.children[0];
  if (!firstChild) return JSON.stringify({drift: true, reason: 'Grid is empty'});

  var path = [];
  var el = firstChild;
  for (var i = 0; i < 4; i++) {
    if (!el) break;
    path.push({
      tag: el.tagName.toLowerCase(),
      classSnippet: el.className ? el.className.split(' ').slice(0, 3).join(' ') : '',
      childCount: el.children.length
    });
    el = el.children[0];
  }
  return JSON.stringify({domPath: path, gridChildCount: grid.children.length});
})()
```

If `rounded-xl` is gone from the card container, the card styling was refactored. The extraction might still work if `card-title` and price selectors still resolve within the grid children.

## Content Drift

If stale or missing, the flyer hasn't updated yet. Log and skip.

### Schema conformance (run after extraction)

```js
(function(products) {
  var fields = ['name', 'salePrice', 'regularPrice', 'size', 'saveAmount'];
  var report = {};
  fields.forEach(function(f) {
    var nullCount = products.filter(function(p) { return !p[f]; }).length;
    report[f] = {
      nullCount: nullCount,
      nullPct: Math.round((nullCount / products.length) * 100)
    };
  });

  // Price format validation
  var pricePattern = /^\$\d+\.?\d*$/;
  var badPrices = products.filter(function(p) {
    return p.salePrice && !pricePattern.test(p.salePrice);
  });
  report._badPriceFormat = {count: badPrices.length, samples: badPrices.slice(0, 3).map(function(p) { return p.salePrice; })};
  report._totalProducts = products.length;
  return JSON.stringify(report);
})(products)
```

**Thresholds:**
- `name` null >5%: `.card-title span.text-grey900` broke
- `salePrice` null >15%: `.text-red400` selector changed — check if the color class was renamed
- `regularPrice` null >70%: normal — many items don't show original price
- `size` null >30%: `p.text-sm` is a very generic selector and likely to pick up false positives OR miss targets if the class changes
- `saveAmount` null >40%: `.bg-yellow150` badge class renamed
- `_badPriceFormat` >0: price text format changed (e.g., now includes "ea" or currency code)

## Behavioral Drift

### Grid view toggle verification

After clicking `button.icon-grid`, verify the view actually switched:

```js
(function() {
  // The Flipp iframes should be hidden and the grid should be visible
  var flippIframes = document.querySelectorAll('iframe.flippiframe');
  var flippVisible = [...flippIframes].some(function(f) { return f.offsetParent !== null; });
  var grid = document.querySelector('.grid.grid-cols-1');
  var gridHasProducts = grid && grid.children.length > 0;

  return JSON.stringify({
    flippStillVisible: flippVisible,
    gridExists: !!grid,
    gridHasProducts: gridHasProducts,
    gridProductCount: grid ? grid.children.length : 0,
    viewSwitchWorked: gridHasProducts && !flippVisible
  });
})()
```

**If `viewSwitchWorked: false`:**
- `flippStillVisible: true` + `gridHasProducts: false` → the grid button didn't trigger the switch. Check if the button class changed from `icon-grid`.
- `flippStillVisible: false` + `gridHasProducts: false` → the page is in a blank state. Something more fundamental broke.
- Both true → the grid loaded but the flipbook wasn't hidden. Extraction can probably proceed, but the page is in a weird state.

### Load More verification

After the Load More loop completes, verify the total makes sense:

```js
(function() {
  // Check sidebar category counts
  var catElements = [...document.querySelectorAll('*')].filter(function(el) {
    return /^\w[\w\s&]+\(\d+\)$/.test(el.textContent.trim()) && el.children.length === 0;
  });
  var expectedTotal = 0;
  catElements.forEach(function(el) {
    var m = el.textContent.match(/\((\d+)\)/);
    if (m) expectedTotal += parseInt(m[1]);
  });

  var grid = document.querySelector('.grid.grid-cols-1');
  var loadedCount = grid ? grid.children.length : 0;

  return JSON.stringify({
    sidebarExpectedTotal: expectedTotal,
    loadedProducts: loadedCount,
    loadRatio: expectedTotal > 0 ? Math.round((loadedCount / expectedTotal) * 100) + '%' : 'no_sidebar',
    likelyComplete: expectedTotal > 0 ? loadedCount >= expectedTotal * 0.9 : loadedCount > 100
  });
})()
```

If `loadRatio` is below 80%, the Load More loop terminated early. Possible causes: button text changed, new anti-bot throttling, or the page switched to a different pagination mechanism.

## FreshCo-Specific Drift Risks

FreshCo (owned by Empire/Sobeys) has the **highest drift risk** of the three stores because:

1. **Tailwind utility classes are generated, not semantic.** A Tailwind config update (e.g., changing `red400` to `red-500`) breaks price selectors without any visible change to the user.
2. **`p.text-sm` is extremely generic.** This could match any small text on the page, not just size info. If the page adds more `text-sm` elements, the size extraction could start pulling in wrong data without returning null (silent corruption).
3. **`#Scene_Card_exclusive_offers` is an ID used on multiple elements** (which is technically invalid HTML). If they fix this to use unique IDs or switch to classes, the selector breaks.
4. **The Flipp flipbook is a third-party integration.** If Sobeys drops Flipp or Flipp changes their iframe class names, the flipbook detection breaks. The grid view would still work, but the skill's logic for detecting whether the flipbook is hidden would need updating.
